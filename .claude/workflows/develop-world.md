# 自动化开发工作流：按世界推进开发

> 每次开发新世界时，按此流程执行。

## Phase 1: 阅读规范
- 读取 `docs/GAME_DESIGN.md` 获取目标世界的完整设计
- 读取 `docs/ARCHITECTURE.md` 确认技术架构
- 读取 `CLAUDE.md` 确认当前阶段和约束

## Phase 2: 分析现有代码
- 扫描 `src/` 目录确认当前代码状态
- 确认哪些模块需要修改、哪些需要新建
- 列出开发任务清单

## Phase 3: 实现开发
按以下顺序实现（每个世界）：

### 3.1 关卡设计
- 在 `LevelManager` 中添加新关卡数据
- 设计瓦片地图布局
- 设置生成点和边界

### 3.2 新实体
- 根据 GAME_DESIGN 创建该世界的 Boss 实体
- 创建该世界的专属敌人（如有）
- 创建该世界的 NPC 实体

### 3.3 新机制
- 实现该世界的核心玩法机制
- 扩展 CollisionSystem（如需要）
- 扩展 Player 能力（如需要）

### 3.4 收集物
- 实现语言碎片收集逻辑
- 实现 Bug 日志（隐藏收集）

### 3.5 视觉效果
- 该世界的配色方案
- 瓦片绘制样式
- 特效（粒子、动画）

## Phase 4: 集成测试
- 在浏览器中验证可运行
- 测试碰撞、Boss 行为、收集逻辑
- 确认无控制台错误

## Phase 5: 更新文档
- 更新 `GAME_DESIGN.md` 中对应世界的开发状态
- 更新 `CLAUDE.md` 中的当前阶段
- 记录实现中发现的设计调整

## 通用开发检查清单
- [ ] 读取 GAME_DESIGN.md 确认目标
- [ ] 确认当前代码状态
- [ ] 实现关卡数据
- [ ] 实现新实体（Boss、敌人、NPC）
- [ ] 实现新机制
- [ ] 实现收集物
- [ ] 视觉效果
- [ ] 浏览器测试
- [ ] 更新文档状态

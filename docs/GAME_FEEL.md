# 游戏手感规范 (Game Feel Specification)

> 参考：Celeste (Matt Thorson)、Vlambeer "The Art of Screenshake"、Steve Swink《Game Feel》

## 核心理念
手感 > 视觉。玩家感受操控是否"跟手"比画面好不好更重要。

---

## 1. 移动参数

### 水平移动
| 参数 | 值 | 说明 |
|------|-----|------|
| 最大速度 | 200 px/s | 满速 |
| 加速度 | 1200 px/s² | 从静止到满速约 0.17s |
| 减速度（松手） | 1600 px/s² | 停止比启动更快，操控感更强 |
| 空中加速度 | 800 px/s² | 空中操控略弱于地面 |
| 转向加速度 | 2000 px/s² | 反向按键时快速转向 |

### 实现方式
```javascript
// 不要直接设 vx = speed，用加速度
if (input.right) {
  vx = Math.min(vx + accel * dt, maxSpeed);
} else if (input.left) {
  vx = Math.max(vx - accel * dt, -maxSpeed);
} else {
  // 减速到 0
  if (vx > 0) vx = Math.max(0, vx - decel * dt);
  else vx = Math.min(0, vx + decel * dt);
}
```

## 2. 跳跃手感

### 基础参数
| 参数 | 值 | 说明 |
|------|-----|------|
| 跳跃初速度 | -450 px/s | 向上 |
| 重力 | 980 px/s² | 下落 |
| 短按重力倍率 | 2.5x | 松开跳跃键时重力增大，实现高低跳 |
| 最大下落速度 | 600 px/s | 终端速度，防止穿透 |
| 二段跳速度 | -380 px/s | 比一段跳略弱 |

### Coyote Time（土狼时间）
```
离开平台边缘后 100ms 内仍可跳跃。
实现：记录 grounded 变为 false 的时间戳，100ms 内 jump 仍然有效。
```
| 参数 | 值 |
|------|-----|
| 土狼时间窗口 | 100ms |

### Jump Buffering（跳跃缓冲）
```
落地前 100ms 内按跳跃键，落地瞬间自动起跳。
实现：记录按跳跃键的时间戳，落地时检查是否在窗口内。
```
| 参数 | 值 |
|------|-----|
| 跳跃缓冲窗口 | 100ms |

### 变量高度跳跃
```javascript
// 按住跳跃键：完整跳跃
// 短按跳跃键：小跳（松开时增大重力）
if (!input.jump && vy < 0) {
  vy += gravity * (shortHopMultiplier - 1) * dt;
}
```

### Corner Correction（角落校正）
```
跳跃时如果头顶刚好擦到平台边缘，自动水平偏移 1-4px 帮助玩家通过。
避免"明明看起来能过却被卡住"的挫败感。
```
| 参数 | 值 |
|------|-----|
| 校正距离 | 4px（每侧）|
| 校正方向 | 向较少阻挡的一侧 |
| 仅触发于 | 跳跃上升阶段 + 头部碰撞 |

```javascript
// 头部碰撞时，尝试左右偏移
if (headCollision && !checkCollision(x + CORNER_CORRECTION, y)) {
  x += CORNER_CORRECTION; // 向右校正
} else if (headCollision && !checkCollision(x - CORNER_CORRECTION, y)) {
  x -= CORNER_CORRECTION; // 向左校正
}
```

### 碰撞箱收紧（Hitbox Tightening）
```
碰撞箱比精灵视觉尺寸略小（每边 2-4px），减少"看起来没碰到但判定命中"的问题。
```
| 对象 | 精灵尺寸 | 碰撞箱 | 收紧量 |
|------|---------|--------|--------|
| 玩家 | 32×48 | 28×44 | 每边 2px |
| 敌人 | 32×32 | 28×28 | 每边 2px |
| 收集物 | 20×20 | 24×24 | 反向放大（更容易收集）|

## 3. Squash & Stretch（形变）

| 动作 | 效果 | 参数 |
|------|------|------|
| 起跳 | 拉伸 Y 1.2x，压缩 X 0.85x | 持续 0.1s，弹性恢复 |
| 落地 | 压缩 Y 0.7x，拉伸 X 1.3x | 持续 0.08s，弹性恢复 |
| 着地前（空中） | 轻微拉伸 Y 1.05x | 根据下落速度动态调整 |

### 实现
```javascript
// 渲染时应用形变
ctx.save();
ctx.translate(centerX, centerY);
ctx.scale(scaleX, scaleY);
ctx.translate(-centerX, -centerY);
// 绘制角色
ctx.restore();

// scaleX/scaleY 用弹性插值恢复到 1.0
scaleX += (1.0 - scaleX) * 0.2;
scaleY += (1.0 - scaleY) * 0.2;
```

### 屏幕震动
| 触发 | 强度 | 持续时间 | 衰减 |
|------|------|---------|------|
| 踩死敌人 | 3px | 0.15s | 线性衰减 |

| 玩家受伤 | 4px | 0.2s | 线性衰减 |
| Boss 登场 | 10px | 0.5s | 指数衰减 |

### 相机行为
| 行为 | 说明 |
|------|------|
| 前瞻偏移 | 角色朝向方向偏移 50px |
| 垂直死区 | 垂直移动超过 30px 才跟随 |
| 落地压缩 | 落地瞬间相机 Y 轴下压 5px |

## 5. 命中反馈 (Hit Feedback)

| 事件 | 视觉 | 音效 | 暂停 |
|------|------|------|------|
| 收集金币 | 粒子爆发 + 弹出数字 "+10" | 叮叮声 | 无 |
| 踩死敌人 | 敌人扁平化 + 粒子 | 咯吱声 | 帧冻结 30ms |
| 玩家受伤 | 红色闪烁 0.5s + 后退 | 受伤声 | 帧冻结 50ms |
| Boss 被击中 | 白色闪烁 + 震屏 | 沉闷撞击声 | 帧冻结 80ms |
| 玩家死亡 | 灰度化 + 粒子爆炸 | 死亡音效 | 帧冻结 200ms |

### 帧冻结 (Hit Stop)
```javascript
// 在 Game.update() 中
if (this.hitStopTimer > 0) {
  this.hitStopTimer -= realDt;
  return; // 跳过物理更新，但继续渲染
}
```

## 7. 调参建议

### 调参原则
1. **先硬编码，后数据驱动**：初期参数写死，跑通后再抽到配置文件
2. **运行时可调**：开发模式下用快捷键打开调参面板
3. **录屏对比**：录下操作视频，逐帧检查手感
4. **找人试玩**：观察别人的操作习惯，发现手感问题

### 开发模式快捷键
| 快捷键 | 功能 |
|--------|------|
| F1 | 显示碰撞框 |
| F2 | 显示速度/加速度数值 |
| F3 | 显示 FPS |
| F4 | 打开调参面板 |
| F5 | 无敌模式 |
| F6 | 传送至检查点 |

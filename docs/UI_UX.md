# UI/UX 设计规范 (UI/UX Specification)

## 核心理念
UI 服务于游戏体验，不打断玩家心流。信息层级清晰，反馈即时。

---

## 1. HUD 布局

### 屏幕分区
```
┌─────────────────────────────────────────────────┐
│ [❤❤❤]  HP          世界1-1        [🪙 120]    │  顶部栏
│                                                 │
│                                                 │
│                  游戏画面                        │  游戏区域
│                                                 │
│                                                 │
│  [碎片 2/3]                       [小地图]      │  底部栏
└─────────────────────────────────────────────────┘
```

### HUD 元素定义

| 元素 | 位置 | 锚点 | 内容 | 字号 |
|------|------|------|------|------|
| 生命值 | 左上 (16, 16) | 左上 | 爱心图标 × N | 16px |
| 关卡名 | 顶部居中 | 顶部中 | "世界1-1 机械之心" | 14px |
| 金币数 | 右上 (-16, 16) | 右上 | "🪙 120" | 16px |
| 碎片进度 | 左下 (16, -16) | 左下 | "碎片 2/3" | 14px |
| 小地图 | 右下 (-16, -16) | 右下 | 120×80 缩略图 | — |
| Boss 血条 | 顶部居中 | 顶部中 | Boss 战时显示 | — |

### HUD 显示规则
- **探索模式**：显示全部 HUD
- **Boss 战**：隐藏碎片进度和小地图，显示 Boss 血条
- **过场动画**：隐藏所有 HUD
- **受伤闪烁**：生命值图标闪烁 0.5s

## 4. 无障碍设计

### 色盲模式
| 模式 | 说明 | 实现 |
|------|------|------|
| 正常 | 默认配色 | — |
| 红绿盲 (Deuteranopia) | 避免红绿同时出现 | 全局滤镜 + 替换配色 |
| 蓝黄盲 (Protanopia) | 避免蓝黄同时出现 | 全局滤镜 + 替换配色 |
| 全色盲 | 高对比度黑白 | 灰度 + 图案区分 |

### 字体大小
| 级别 | 字号 | 适用 |
|------|------|------|
| 标准 | 14-16px | 默认 |
| 大 | 18-20px | 视力不佳 |
| 超大 | 24-28px | 无障碍 |

### 按键重映射
- 支持自定义所有操作的按键绑定
- 预设方案：WASD / 方向键 / 手柄
- 显示当前绑定的按键图标

### 其他无障碍
| 特性 | 说明 |
|------|------|
| 高对比度 UI | 文字与背景对比度 ≥ 4.5:1 (WCAG 1.4.3) |
| 屏幕阅读器 | 菜单支持 ARIA 标签 |
| 震动手柄 | 支持手柄震动反馈 |
| 自动瞄准 | 可选的辅助功能 |
| 慢动作模式 | 游戏速度 0.5x |
| 减少闪烁 | 屏幕震动和闪烁可关闭（光敏性癫痫防护）|
| 字幕系统 | NPC 对话显示字幕，可调大小和背景 |
| 音频提示替代 | 重要信息不只靠声音传达（视觉+音效双通道）|

## 5. 动画规范

### UI 动画时长
| 动画 | 时长 | 缓动函数 |
|------|------|---------|
| 菜单淡入 | 0.3s | ease-out |
| 菜单淡出 | 0.2s | ease-in |
| 弹窗滑入 | 0.3s | ease-out-back |
| 弹窗滑出 | 0.2s | ease-in |
| 按钮悬停 | 0.1s | linear |
| 按钮按下 | 0.05s | ease-in |
| 数字滚动 | 0.5s | ease-out |
| 进度条填充 | 0.8s | ease-in-out |

### 缓动函数
```javascript
const Easing = {
  linear: t => t,
  easeIn: t => t * t,
  easeOut: t => t * (2 - t),
  easeInOut: t => t < 0.5 ? 2*t*t : -1+(4-2*t)*t,
  easeOutBack: t => { const s = 1.70158; return 1 + (s+1) * Math.pow(t-1, 3) + s * Math.pow(t-1, 2); },
};
```

## 6. 响应式布局

### Canvas 缩放策略
```javascript
function resizeCanvas() {
  const ratio = 16 / 9;
  let w = window.innerWidth;
  let h = window.innerHeight;

  if (w / h > ratio) {
    w = h * ratio;
  } else {
    h = w / ratio;
  }

  canvas.style.width = `${w}px`;
  canvas.style.height = `${h}px`;
  // 内部分辨率保持 960×540
}

window.addEventListener('resize', resizeCanvas);
```

## 7. 通知系统

### 通知类型
| 类型 | 显示位置 | 持续时间 | 样式 |
|------|---------|---------|------|
| 成就 | 顶部居中 | 3s | 金色边框 |
| 收集 | 底部居中 | 2s | 对应颜色 |
| 提示 | 中部 | 直到操作 | 半透明背景 |
| 错误 | 中部 | 1.5s | 红色闪烁 |

### 通知队列
- 最多同时显示 3 条通知
- 新通知从底部推入，旧通知上移
- 超出数量的旧通知立即消失

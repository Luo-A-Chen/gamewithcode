# 难度与进度设计规范 (Difficulty & Progression Specification)

## 核心理念
难度服务于学习。每个新机制都有"安全练习区"，然后逐步叠加挑战。

---

## 1. 教学节奏模型

### 三阶段教学法
```
引入 → 练习 → 考验
  │       │       │
  │       │       └── 组合多个机制的挑战
  │       └── 在安全环境中反复练习
  └── 无压力环境展示新机制
```

### 每个世界的教学流程
| 阶段 | 关卡 | 设计目标 |
|------|------|---------|
| 引入 | 第 1 小节 | 无敌人，只有新机制，失败无惩罚 |
| 练习 | 第 2-3 小节 | 有少量敌人，新机制是主要挑战 |
| 考验 | 第 4 小节 | 组合所有学到的机制 |
| Boss | Boss 战 | 综合运用本世界所有技能 |

## 3. 生命系统

### 方案：血量 + 命

```
HP（生命值）：每条命有 3 点 HP
命（Lives）：初始 3 条命
```

### 伤害来源
| 来源 | 伤害 | 无敌时间 | 击退力 |
|------|------|---------|--------|
| 接触普通敌人 | 1 HP | 1.5s | 向后 200px/s |
| 接触精英敌人 | 2 HP | 1.5s | 向后 300px/s |
| 掉入尖刺 | 2 HP | 2.0s | 向上 400px/s |
| Boss 攻击 | 2 HP | 2.0s | 向后 400px/s |
| 掉出世界 | 即死 | — | 重生 |

### 恢复方式
| 方式 | 恢复量 | 来源 |
|------|--------|------|
| 小血瓶 | 1 HP | 敌人掉落 / 隐藏宝箱 |
| 大血瓶 | 全部 HP | 关卡末尾 |
| 1UP | +1 命 | 隐藏收集 |
| 检查点 | 全部 HP + 命 | 到达检查点 |

### 死亡惩罚
```
HP 归零 → 消耗 1 命 → 从当前检查点重生（HP 全满）
命归零 → Game Over → 从关卡开头重新开始（保留收集物）
```

## 4. 敌人设计

### 敌人类型谱系

| 类型 | 行为 | 难度 | 首次出现 |
|------|------|------|---------|
| 巡逻兵 | 固定路线来回走 | ★ | 世界1 |
| 飞行兵 | 上下/左右飞行 | ★★ | 世界2 |
| 射手 | 定时发射弹幕 | ★★★ | 世界3 |
| 精英 | 多 HP、攻击模式 | ★★★★ | 世界4 |
| 护盾兵 | 正面无敌，需绕后 | ★★★ | 世界5 |
| 分裂怪 | 死后分裂成小怪 | ★★★★ | 世界6 |
| 瞬移怪 | 随机瞬移 | ★★★★★ | 世界7 |

### 敌人配置公式
```
关卡敌人数量 = 基础值 + (世界编号 × 2) + (关卡小节 × 1)
敌人速度 = 基础速度 + (世界编号 × 15) + (关卡小节 × 5)
```

## 5. Boss 设计

### Boss 通用框架

| 阶段 | 行为 | 条件 |
|------|------|------|
| Phase 1 | 1-2 种攻击模式 | HP > 66% |
| Phase 2 | 增加 1 种攻击模式，速度提升 20% | HP 33%-66% |
| Phase 3 | 所有攻击模式，速度再提升 20% | HP < 33% |

### 各世界 Boss 设计

#### 世界1：被卡住的差分机
- **Phase 1**：齿轮旋转攻击（固定路线）
- **Phase 2**：齿轮 + 蒸汽喷射（范围攻击）
- **Phase 3**：齿轮加速 + 蒸汽 + 掉落齿轮
- **弱点**：背后的核心（需绕后攻击）

#### 世界2：暴走的继电器
- **Phase 1**：电弧攻击（直线）
- **Phase 2**：电弧 + 电场（圆形范围）
- **Phase 3**：连锁电弧 + 电场 + 闪电链
- **弱点**：过载时暴露的线路

#### 世界3：无限递归怪物
- **Phase 1**：分身攻击（3 个分身）
- **Phase 2**：分身 + 弹幕（递归弹幕）
- **Phase 3**：无限分身（需找到"递归出口"机关）
- **弱点**：递归终止条件（踩按钮）

#### 世界4：野指针幽灵
- **Phase 1**：瞬移 + 冲刺攻击
- **Phase 2**：瞬移 + 残影攻击
- **Phase 3**：瞬移 + 残影 + 范围爆炸
- **弱点**：瞬移后的短暂硬直

#### 世界5：内存泄漏黑洞
- **Phase 1**：缓慢膨胀 + 吸引力
- **Phase 2**：膨胀加速 + 释放碎片
- **Phase 3**：巨大化 + 全屏引力 + 碎片风暴
- **弱点**：膨胀到最大时的短暂缩小

#### 世界6：XSS 攻击虫群
- **Phase 1**：小虫群冲锋
- **Phase 2**：虫群 + 注入攻击（变大）
- **Phase 3**：虫群分裂 + 注入 + 脚本攻击
- **弱点**：核心虫后（消灭虫群后暴露）

#### 世界7：Bug King（最终 Boss）
- **Phase 1**：使用世界1-2的攻击模式
- **Phase 2**：使用世界3-4的攻击模式
- **Phase 3**：使用世界5-6的攻击模式
- **Phase 4**：混合所有攻击 + 终极技能
- **弱点**：每个阶段对应历史时代的弱点

## 6. 进度门控

### 世界解锁条件
| 世界 | 解锁条件 |
|------|---------|
| 1 | 默认 |
| 2 | 通关世界1 |
| 3 | 通关世界2 + 收集 2 个碎片 |
| 4 | 通关世界3 + 收集 4 个碎片 |
| 5 | 通关世界4 + 收集 7 个碎片 |
| 6 | 通关世界5 + 收集 10 个碎片 |
| 7 | 通关世界6 + 收集 14 个碎片 |

### 碎片总览
- 每个世界 3 个碎片
- 总共 21 个碎片
- 通关所有世界需要 14 个（67%）
- 全收集需要探索所有隐藏区域

## 7. 评价系统

### 关卡评价
| 指标 | ⭐ | ⭐⭐ | ⭐⭐⭐ |
|------|-----|------|--------|
| 通关时间 | < 180s | < 120s | < 60s |
| 收集物 | ≥ 50% | ≥ 80% | 100% |
| 死亡次数 | < 10 | < 5 | 0 |

### 评价计算
```
总星数 = 时间星数 + 收集星数 + 死亡星数
3星 = 全部 ⭐⭐⭐
2星 = 至少2项 ⭐⭐
1星 = 通关即可
```

### 奖励
| 评价 | 奖励 |
|------|------|
| ⭐ | 解锁下一关 |
| ⭐⭐ | 解锁隐藏区域 |
| ⭐⭐⭐ | 解锁图鉴条目 |

## 8. 动态难度调节（可选）

### 自适应机制
如果玩家在同一关卡死亡超过 5 次：
- 敌人速度降低 10%
- 检查点增加一个
- 提示文字出现

### 调节参数
```javascript
const adaptiveDifficulty = {
  deathThreshold: 5,      // 触发调节的死亡次数
  speedReduction: 0.1,    // 速度降低比例
  extraCheckpoint: true,  // 是否添加额外检查点
  showHints: true,        // 是否显示提示
};
```

## 9. 辅助模式 (Assist Mode)

> 参考：Celeste 的 Assist Mode，让所有人都能体验完整游戏。

### 设计原则
- 辅助模式是**可选的**，不影响正常玩家
- 开启后**不标记为作弊**，不降低评价
- 目标：让身体受限的玩家也能通关

### 可调参数
| 参数 | 选项 | 说明 |
|------|------|------|
| 游戏速度 | 100% / 75% / 50% | 整体减速 |
| 无敌模式 | 开/关 | 不受伤害 |
| 无限跳跃 | 开/关 | 空中可无限跳 |
| 跳跃高度 | 100% / 125% / 150% | 跳得更高 |
| 冲刺次数 | 1 / 2 / 3 | 额外冲刺（世界4+）|
| 自动前进 | 开/关 | 自动向右移动 |

### 入口
```
设置 → 辅助模式 → [确认开启] → 调整参数
```
- 首次死亡 3 次后，显示提示："是否需要开启辅助模式？"
- 可随时开关，不影响存档

## 10. 新手引导

### 引导方式
| 方式 | 适用场景 | 示例 |
|------|---------|------|
| 文字提示 | 操作说明 | "按空格跳跃" |
| 视觉引导 | 路线提示 | 发光的路径 |
| 安全练习 | 新机制 | 无敌人环境 |
| NPC 对话 | 故事/背景 | 巴贝奇讲解 |

### 提示文字规则
- 首次出现时显示，之后不再显示
- 按任意键跳过
- 不打断玩家操作
- 位置：屏幕底部居中，半透明背景
